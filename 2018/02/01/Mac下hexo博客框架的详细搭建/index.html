Mac下hexo博客框架的详细搭建

发表于 2018-02-01 | 分类于 教程 ， Mac

hexo的框架详细使用方法，带guthub配置

引用作者：https://lfkid.github.io/

一、安装 Hexo

在命令行输入以下命令，以将 Hexo 安装到全局环境：

npm install hexo-cli -g

成功安装后，会出现诸如 -- hexo-cli@1.0.2 的提示信息。

注意！提示信息末尾可能会出现 npm WARN 警告信息，不影响使用。

二、初始化站点文件夹

接下来，我们将运行命令初始化一个新的文件夹，作为我们的本地站点：

hexo init hexo

以上命令将先在当前路径下新建一个名为 hexo 的文件夹，然后 Hexo 会对其初始化，即从远程 git 仓库 clone 站点所需的文件。

接着，进入新建的站点文件夹 hexo，继续安装站点文件所需的 npm 包依赖：

cd hexo
npm install

安装服务器

Hexo 3.0 把服务器独立成了个别模块，我们必须得在站点文件夹根目录下安装之后才能使用。

在命令行中打开我们的 hexo 文件夹，键入以下命令，以将 hexo-server 安装到我们的 hexo 文件夹：

npm install hexo-server --save

Hexo基础命令

* 新建文档

假设我们的文章名为 “hello hexo”，在命令行键入以下命令即可：

hexo new "hello hexo"

上述命令的结果是在 ./hexo/source/_posts 路径下新建了一个 hello-hexo.md 文件。

然后用本地markdown编辑器打开编辑保存即可，想要删除就直接删除。

注意：

categories 有一定的相关性、顺序性和层次性。

简单来说，categories 有点儿像新建文件夹对文档进行分门别类的归置。

categories 的用法同 tags 一样

* 清除缓存文件

hexo clean

此命令不可以简写。

此命令用于清除缓存文件（db.json）和已生成的静态文件(public)。

* 生成静态文件

hexo generate

此命令用于生成静态文件，就是按照主题的模板文件配置将 markdown 文件生成为静态 html 文件。

此命令可简写为 hexo g。

当运行上述命令后，我们站点文件夹的目录结构下会多出一个 public 文件夹，这个文件夹中的文件就是 Hexo 帮我们生成的站点静态文件。

* 启动服务器

hexo server

此命令会启动 Hexo 服务器，可简写为 hexo s。

默认状态下，我们的网站是运行在 http://localhost:4000 ，使用浏览器打开这个地址就可看到效果。

配置文件_config.yml

Hexo 所初始化的站点文件夹根目录下的 ._config.yml 文件声明了网站的配置信息，我们可以通过修改此文件的配置参数以个性化网站。

该文件位于站点文件夹下的根目录下。我们可以通过修改 ._config.yml 文件中的参数配置来自定义 Hexo 生成的静态站点。

使用文本编辑器打开该文件，我们会发现其中有详尽的参数，这里只选取一部分具有代表性的参数配置进行介绍。详情戳这里 ⇨ 配置。

* 网站相关

  • title ：网站标题
  • subtitle ：网站副标题
  • description ：网站描述
  • author ：作者名字
  • language : zh-Hans

* 网址相关

  • url ：网址
  • root ：网站根目录

* 文章相关

  • auto_spacing ：在中文和英文之间加入空格，默认值为 false
  • external_link ：在新标签中打开链接，默认值为 true
  • render_drafts ：显示草稿，默认值为 false
  • post_asset_folder ：开启文章地资源管理文件夹，默认值为 false
  • highlight ：代码块的设置，包括有 enable、line_number、auto_detect 和 tab_replace 属性可设置。不过一般不用修改，大多数主题都默认是支持代码语法高亮等设置的。

* 分页相关

  • per_page ：每页显示的文章数，默认值为 10，值为 0 时会关闭分页功能

新建菜单页

Hexo 生成的站点默认菜单也有限，如果我们想自定义添加菜单页该怎么操作呢？

  • 添加About

hexo new page "about"

上面的命令生效以后，根目录下的 source 文件夹中会新增一个名为 about 的文件夹，里面有个 index.md 文档。我们将想要在 About 菜单页中显示的内容，按照博文格式写在这个文档里即可。

然后，修改 ./themes/your-theme-name/_config.yml 文件中的 menu 项，在下面添加一行 About: /about 即可。

  • 添加标签

hexo new page tags

确认站点配置文件里有tag_dir: tags
确认主题配置文件里有tags: /tags
编辑站点的source/tags/index.md，添加

  • 添加分类

hexo new page categories

确认站点配置文件里有category_dir: categories
确认主题配置文件里有categories: /categories
编辑站点的source/categories/index.md，添加

将本地Hexo站点部署到GitHubPages

* git和ssh的安装，见我另外一篇博客<>

* 创建github.io仓库

登入你自己的 GitHub 账户，创建一个新的仓库，仓库名为 username.github.io，其中 username 替换为你自己的 GitHub 用户名，就是你个人 GitHub 主页 URL 中的最后一部分。
创建好仓库后，做一次初始提交，以便初始化这个仓库。

* 配置 SSH Pub Key

打开你的 GitHub 个人主页，点击右上角个人头像，选择 Settings，进入设置页面。
在左侧选择 SSH and GPG keys，然后在右侧点击 New SSH key，将刚刚创建的 SSH Key 的公钥文件 id_rsa.pub 中的内容复制到 Key 文本框中。

* 本地安装 Hexo 插件 hexo-deployer-git

在站点文件夹根目录下打开命令行，键入以下命令：

npm install hexo-deployer-git --save

  • message ：自定义 Git 的本次 commit 信息（Hexo会按照一定默认生成）

* 部署

如果我们已经生成好了本地静态站点，那么只需下面的命令，即可将其一键部署到服务器上。使用安装好的 Git Bash 终端，在站点根目录下键入：

hexo deploy

Windows用户注意 以上命令一定要在 Git Bash 终端中键入，因为我们使用的 SSH 协议推送本地仓库到远程分支，必需使用到 Git Bash 内置的 Open SSH 客户端。在此期间，会跳出 Open SSH 的界面，让我们输入密码，就是之前我们用来加密 SSH Key 的在命令行输入的那个密码。
